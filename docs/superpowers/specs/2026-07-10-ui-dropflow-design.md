# TempoGraph UI v3 — "Drop → Watch → Explore" design

Date: 2026-07-10 · Author: brainstormed with Claude, approved by Aswin
Executor: local Claude Code (qwen 35B) overnight, branch `ui-v3-dropflow`

## Problem

The current Streamlit UI (`ui/app.py`, 750 lines) front-loads ~20 expert
knobs across six sidebar sections before the user has even chosen a video.
The pipeline underneath already auto-detects hardware, auto-starts services,
and estimates runtimes — the UI just doesn't use any of that to help the
user. Target experience: **drop a video in, get everything.**

## Design

Three screens, replacing the knob wall:

### 1. Landing
- Full-width video drop zone (`st.file_uploader`) + a "path or URL" text
  fallback. No sidebar on this screen.
- Below: recent-runs gallery from the existing SQLite stores (thumbnail,
  date, entity/event/caption counts). Click → opens that run's Results.

### 2. Auto-plan
- On drop, `ffprobe` reads duration / resolution / audio presence.
- A new `src/auto_profile.py` derives every current knob from those facts:
  - YOLO variant: short clip (≤3 min) → `yolo26x-seg`, else `yolo26n-seg`;
    sweep FPS budgeted by duration; confidence stays 0.5.
  - Whisper: skipped when no audio track; `base.en` otherwise.
  - VLM captioning: frame rate fitted to a time budget via
    `runtime_estimator`; dynamic chunking on; dedup default.
  - Depth: on.
- UI shows ONE sentence (the plan + ETA) + one primary **Analyze** button +
  one collapsed **Adjust plan** expander containing the existing knobs,
  pre-filled with the derived values. Advanced users lose nothing.

### 3. Progress → Results
- Progress: stage checklist (queued/running/done), per-stage ETA, live
  counters (entities/captions so far), a working **Cancel** button
  (TODO.md item: `threading.Event` into `PipelineV2.run()`).
- Stage failure = warning banner + continue ("audio failed — everything
  else complete"), never a traceback in the UI.
- Results dashboard (auto-opens): tabs
  - **Overview** — stat cards + 5-line narrative summary generated by the
    already-warm qwen from aggregated captions + transcript
  - **Timeline** — captions + transcript aligned on a scrubber with
    keyframe thumbnails
  - **Graph** — existing graph view
  - **Ask** — free-text Q&A grounded in the run's SQLite
  - **Export** — dataset exporter wired in (TODO.md Tier-1 item)
